__vmalloc_area_node

2017-02-24

1.__vmalloc_node()

__vmalloc_node -> __vmalloc_node_range
 *  Allocate enough pages to cover @size from the page level
 *  allocator with @gfp_mask flags.  Map them into contiguous
 *  kernel virtual space, using a pagetable protection of @prot.
 所以, 我们需要
1. 找一个可用的虚拟地址空间
2. 把 够分量的@size, 可以物理地址上非连续的, 符合@gfp_mask的 page 分配出来
3. 虚拟地址和 物理地址 关联上, 并设置相应的 权限@prot.

__vmalloc_node_range
|--struct vm_struct *area=__get_vm_area_node
|--__vmalloc_area_node
|   |--alloc_page
|   |--map_vm_area

map_vm_area()将struct vm_struct和struct page关联起来.

map_vm_area ==> vmap_page_range
|--vmap_page_range_noflush  改写页表, 但如函数名所说的, 不执行flush
|--flush_cache_vmap

vmap_page_range_noflush
|...
|--vmap_pte_range



__vmalloc_area_node
{
    /*问题:为何这里是 减去PAGE_SIZE ? */
    nr_pages = (area->size - PAGE_SIZE) >> PAGE_SHIFT;
    ...

}

问题:为何这里是 减去PAGE_SIZE ?
答:

__get_vm_area_node
{
    size = PAGE_ALIGN(size);
    /*
     * We always allocate a guard page.
     */
    size += PAGE_SIZE;

    alloc_vmap_area(size, ...);
}

所以, 减去的是 guard page.
所以, 作为guard的page是虚拟地址是占用了, 不过page frame就不必分配了, 反正我们不是想存到guard page中去的.

2. 递归

__vmalloc_area_node
{

    array_size = (nr_pages * sizeof(struct page *)); 

    if (array_size > PAGE_SIZE) {
        pages = __vmalloc_node(array_size, 1, nested_gfp|__GFP_HIGHMEM,
                PAGE_KERNEL, node, caller);
        area->flags |= VM_VPAGES;
    } else {
        pages = kmalloc_node(array_size, nested_gfp, node);
    }
}

这里分配的内存是用来存放 成员为struct page *的数组. 我们很爱惜物理地址连续的内存, 如果 用来存储struct page *的超过一个物理页, 那么, 我们还是坚持尽量从order==0中来分配.

这里递归了, 因为

vmalloc
|--__vmalloc_node_flags -> __vmalloc_node -> __vmalloc_node_range
|   |--__get_vm_area_node
|   |--__vmalloc_area_node
|   |   |--__vmalloc_node
|   |   |   |--kmalloc_node

__vmalloc_node()调用了 __vmalloc_node().

释放中对应地, 也有递归

__vunmap
{
    if (area->flags & VM_VPAGES)
        vfree(area->pages); //这里是递归 (先释放里面的, 再释放外面的)
    else
        kfree(area->pages);

}
